### [x] Task 1.6: Create RV Verification Controller

**Objective**: Implement business logic for all RV verification operations.

**Contract Reference**: Section "3. API Endpoints" (3.1 - 3.4) in spec.md

**Implementation Steps**:
1. Create file `backend/controllers/rvVerificationController.js`
2. Import dependencies: User, RVVerification models, emailService
3. Implement `exports.startVerification` (POST /start):
   - Validate RV email against regex: `/^[^\s@]+@(rvce\.edu\.in|rv\.edu\.in|ms\.rvce\.edu\.in)$/i`
   - Generate 6-digit OTP: `Math.floor(100000 + Math.random() * 900000).toString()`
   - Set `otpExpiresAt` to `new Date(Date.now() + 10 * 60 * 1000)`
   - Upsert RVVerification record using `findOneAndUpdate` with `upsert: true`
   - Send email via `emailService.sendRVVerificationOTP()`
   - Return success response
4. Implement `exports.verifyOTP` (POST /verify-otp):
   - Find RVVerification by `req.user.id`
   - Check OTP matches and not expired
   - If valid: set `emailVerified = true`, `status = 'verified'`, `verifiedAt = new Date()`
   - Clear OTP fields
   - Return success with verification data
5. Implement `exports.getStatus` (GET /status):
   - Find RVVerification by `req.user.id`
   - If not found: return `{ rvVerified: false, status: 'none' }`
   - If found: return appropriate response based on status
6. Implement `exports.adminReview` (POST /admin-review):
   - Find RVVerification by `userId` from request body
   - Validate status change (cannot verify if emailVerified = false)
   - Update status, set verifiedAt or rejectedAt
   - Update notes
   - Return updated verification record

**Deliverable**: `backend/controllers/rvVerificationController.js` file created with 4 exported functions

**Verification**:
